1. Accedendo al sito, e seguendo le indicazioni sulla prima pagina, noto che:
	
    - Accedendo al path http://localhost:8000/?domain=www.ulisse.unibo.it  
    noto che il sito esegue il comando nslookup al dominio www.ulisse.unibo.it
    passato come parametro "domain"
    
2. Da ciò si evince che è possibile sfruttare una vulnerabilità nota: command injection
Essa è una vulnerabilità di sicurezza in cui un'applicazione web accetta input dall'utente 
e lo utilizza all'interno di un comando di sistema senza un'adeguata sanitizzazione o convalida. 
Ciò consentirà ad un attaccante di eseguire comandi arbitrari sul server.

3. Per provare a sfruttare questa vulnerabilità per ottenere il file /tmp/flag_seclab:

	- Provo a concatenare alla variabile il ";" per eseguire un comando successivo
	Il ; è filtrato
	
	- Notando che nslookup fallisce, posso mettere condizione "or" e un comando che verrà 
	eseguito per la suddetta causa. Provo con ls e funziona
	
	http://localhost:8000/?domain= ||ls
