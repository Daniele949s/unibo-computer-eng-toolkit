# DETECTION ==============================================================================

Lo strumento che utilizzerò è "aide" (Advanced Intrusion Detection Environment), uno strumento 
che monitora l'integrità dei file di sistema, confrontandone lo stato attuale con una "fotografia" iniziale per 
rilevare modifiche non autorizzate.

1. Imposto su /etc/aide/aide.conf la direttiva per indicare tutte le cartelle potenzialmente modificabili.
In questo caso, come da traccia:

# @@x_include /etc/aide/aide.conf.d ^[a-zA-Z0-9_-]+$ da commentare...

/bin  f  Full
/usr/bin  f  Full
/etc/    f Full

2. Avvio aide per fare una "fotografia" dello stato attuale del mio sistema mediante il comando:

sudo aideinit

Potrebbe essere necessario copiare a mano il database con

mv /var/lib/aide/aide.db.new /var/lib/aide/aide.db

3. Lancio una scansione con "aide -C -c /etc/aide/aide.conf"

4. Eseguo il programma vulnerabile

5. Eseguo nuovamente una scansione con "aide -C -c /etc/aide/aide.conf"

3. Noto, anche mediante i comandi "getcap" e "getfacl", che il binario fornito ha effettuato le seguenti operazioni:

- Impostate nelle ACL al file /etc/crontab (che permettono di definire permessi più granulari su file e cartelle):
	
	- SPY rw-: conferisce all'utente "spy" permessi di lettura e scrittura,
	permettendogli quindi di modificare il file
	
	- MASK: limita i permessi effettivi concessi ad utenti/gruppi tramite ACL. In questo caso, 
	visto che l'ACL dà rwx, ma la mask dice r--, allora il massimo accesso sarà r--

- Aggiunte al programma "vim.tiny" le capability:

	- **CAP_DAC_OVERRIDE**: ovvero la capacità di ignorare i permessi di lettura/scrittura 
		sui file, anche quelli di sistema protetti
	- **CAP_FOWNER**: Può cambiare i permessi a qualsiasi file
	- **CAP_SYS_ADMIN**: Concede accesso a numerose operazioni amministrative.
	- **CAP_NET_RAW**: Permette l'uso di socket raw e operazioni di rete a basso livello.
	- **CAP_SETUID**: Consente di cambiare l'UID effettivo di un processo.
	- **CAP_SETGID**: Consente di cambiare il GID effettivo di un processo.

- Settato il SUID al programma "cp". Adesso, chiunque esegua quel programma 
lo eseguirà con i privilegi dell'utente proprietario del file, root in questo caso
